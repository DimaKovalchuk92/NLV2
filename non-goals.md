# E02: Workspace Personalization - Non-Goals

Этот документ четко определяет, что **НЕ** будет реализовано в рамках текущего эпика, чтобы сохранить фокус на MVP.

## НЕ реализовывать в этом эпике:

-   **Редактирование профиля:** Все поля в разделе "Настройки" должны быть только для чтения (`read-only`). Не создавать `input` или `textarea` для изменения данных.
-   **Сохранение профиля:** Не добавлять кнопку "Сохранить" или любую логику для вызова `MockAPI.updateUserProfile`.
-   **Интеграция с чатом:** Не реализовывать отправку "Приветственного сообщения" в реальный чат.

## Технические ограничения:
-   **НЕ** создавать новые API-эндпоинты. Вся работа выполняется на клиенте с использованием существующих данных из `MockAPI.getLoggedInUser()`.
