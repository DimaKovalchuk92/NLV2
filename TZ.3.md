0. Визначення продукту
Платформа — це операційна система для онлайн-практики спеціалістів допоміжних професій: психологів, психотерапевтів, коучів, консультантів та ведучих груп/програм. Вона об'єднує:
Безпечну взаємодію клієнт ↔ спеціаліст (відео-сесії, чат, файли, інтерактиви)
CRM та кейс-менеджмент (особові справи, прогрес, версії, аудит)
Маркетплейс спеціалістів, груп, програм та марафонів
Платіжну інфраструктуру з підтримкою страхового білінгу (включаючи контроль оплат, виплати спеціалістам, обробку клеймів)
Інструменти утримання та залучення (прогрес/«Шлях героя», цикли інтеграції, Activity Hub, терапевтичні інструменти між сесіями)
AI-контур (транскрипція, структуровані клінічні нотатки, семантичний пошук, предиктивне флагування ризиків — за згодою)
Мобільні додатки для клієнта та спеціаліста з офлайн-доступом до критичних функцій
Модульну архітектуру для масштабування та майбутніх додатків (alter-ego, wearables, plug-ins)
Мультимовний інтерфейс та контент з повноцінною підтримкою i18n/L10n
Систему онбордингу та UX-супроводу для всіх ролей
Вбудовані механізми доступності (Accessibility, WCAG 2.1 AA, trauma-informed design)
Централізовану систему нотифікацій з гранулярним управлінням
Аналітику та дашборди для спеціалістів, організацій та платформи
Workflow страхових клеймів та крединшнелінгу [CA]
B2B/EAP модуль для контрактів з роботодавцями [CA]
Measurement-based care (MBC) — вимірювання результатів [CA]
Модель асинхронної терапії (messaging-only) [CA]
Модуль для спеціалістів: peer consultation, супервізія, CE кредити [CA]

Система управляє видимістю на рівні об'єктів: який спеціаліст бачить яку особову справу, які частини запису/нотаток доступні клієнту, хто може бути присутнім у груповій активності, хто може експортувати/передавати дані, хто і як використовує AI.

2. Модель даних: Case, Program, Activity
2.1 Case-centric модель (індивідуальна робота)
Case (Особова справа) — зв'язок Client ↔ Specialist у конкретному контексті. Один клієнт може мати багато Case у різних спеціалістів. У кожного Case окремі: цілі, історія, файли, нотатки, згоди, прогрес, версії, intake-дані, outcome-трекінг та страхова інформація.
2.2 Program / Group (групи, курси, марафони)
Program (або Group) — контейнер групового формату: терапевтична група, коучингова програма, марафон, цикл тематичних занять, корпоративна програма. Program має: власника/команду ведучих, формат (live / async / hybrid), правила набору/участі, структуру (цикли, активності), контент і матеріали, чат(и), платіжну модель, метрики залучення, підтримку автоматичної доставки крапельного контенту [CA].
2.3 Activity та Activity Hub
Будь-яка одиниця роботи в рамках Program оформлюється як Activity.
Типи Activity: Core Session (60-120 хв), Integration Standup (10-30 хв), Cycle/Module (контейнер), Closing Session (особлива сутність), Q&A-сесія, майстер-клас, вебінар.
Activity Hub — єдина сторінка активності: Summary, News/Announcements, Chat, Materials, Progress/Reflection.

2.4.1 Передача Case між спеціалістами
Протокол передачі: ініціатор — клієнт або спеціаліст (за згодою клієнта). Передані дані визначаються Consent Engine. Період паралельного доступу: настроюваний (14 днів за замовчуванням). Контекстне саммарі для нового спеціаліста — «тепла передача» з відстеженням статусу рефералу [CA]. Повне логування факту передачі та обсягу даних.
2.4.2 Data retention та експорт
Активні Case: повне зберігання без обмежень.
Archived Case: стиснення, перенос у холодне сховище.
Строки зберігання: per Organization / per Jurisdiction (GDPR — право на видалення; HIPAA — мінімум 6 років).
42 CFR Part 2: сегрегація нотаток SUD counseling, окрема згода [CA].
Психотерапевтичні нотатки: окреме зберігання від клінічних записів за HIPAA [CA].
Експорт: ZIP (JSON + файли + PDF-звіт) у машиночитаному форматі.
Право бути забутим: повне видалення з каскадом. Аудит-лог зберігається окремо.

Контроль оплат (клієнт → спеціаліст, клієнт → платформа, спеціаліст → платформа)
Intake-форми з е-підписами, умовною логікою, автоматизованими послідовностями прийому [CA]
3.2 Асинхронна терапія (messaging-only) [CA] [NEW]
Модель асинхронної терапії — ключовий диференціатор BetterHelp та Talkspace. Платформа повинна підтримувати:
Messaging-only підписку як базовий тариф.
SLA часу відповіді спеціаліста (настроюваний: 2/4/8/24 години).
Мультимедійні повідомлення (текст, аудіо, відео, зображення).
Структуру компенсації спеціалістів за async-роботу (окрема від сесійної).
Індикатор онлайн-статусу та очікуваного часу відповіді.
Метрики залучення: середній час відповіді, кількість обмінів на день.
3.3 Групова терапія / групи підтримки (live)
Типи груп: постійна (фікс. склад), обмежена (cohort-based, 6-12 тижнів), напіввідкрита (додавання за правилами + onboarding-модуль). Групова терапія трактується як first-class модальність зі специфічними інструментами фасилітації [CA]. В групі завжди розділяються два шари: Group Context (правила, матеріали, саммарі) та Individual Layer (особисті нотатки/прогрес — невидимі іншим).
3.4 Програми та марафони (live + async + hybrid)
Структуровані етапи з підтримкою крапельного контенту / evergreen програм [CA]
Live-зустрічі за розкладом
Асинхронні чек-іни
Когортна послідовність — синхронізація груп через контент [CA]
Гейміфікація прогресу
Чати за контекстом
3.5 Циклічна інтеграційна група
Формат із високою прикладною інтеграцією: Core Session (1-2 години) по темі/навичці → серія Integration Standups → наступний цикл → Closing Session. Закриває розрив між «зрозумів на сесії» та «впровадив у житті».

4.2 Grounding — вбудований модуль безпеки
Кнопка «Заземлення» доступна у відеосесії. При активації: у клієнта — візуальний дихальний гайд / анімація / сенсорний протокол; у спеціаліста — панель управління. Режими: індивідуальний, груповий, самостійний. Підтримка Reduced Motion та високого контрасту. Офлайн-доступ у мобільному додатку.
4.3 Кімната очікування / лобі [CA] [NEW]
Стандарт усіх телездоров'я-платформ. Включає: тест камери/мікрофону, перевірку з'єднання, передсесійну діагностику. Зменшує неявки та технічні збої.
4.4 Pre-session флоу
Для клієнта:
За 24 години: email-нагадування з кнопкою підтвердження / перенесення.
За 1 годину: push-повідомлення.
За 15 хвилин: короткий чек-ін (шкала 1-10 + вільне поле) — опціонально.
За 5 хвилин: автоматична перевірка камери, мікрофону, швидкості з'єднання.
Кнопка «Увійти в сесію» — за 5 хвилин до початку.
Для спеціаліста:
Pre-session brief: картка клієнта з останніми нотатками, цілями, статусом домашнього завдання, результатом чек-іну та outcome-трекінгом.
Швидкий доступ до стенографії минулих сесій.
Флаги: маркери risk level, важливі події між сесіями, результати MBC-скринінгу [CA].
4.5 Post-session флоу
Для спеціаліста:
Структуровані шаблони клінічних нотаток (SOAP, DAP, BIRP, GIRP) [CA] з прив'язкою до DSM-5 та SMART-цілями.
AI-суммарі сесії як чернетка у форматі обраного шаблону (якщо AI увімкнено та згоду отримано) [CA].
Призначення домашнього завдання з дедлайном та авто-нагадуванням клієнту.
Призначення терапевтичних інструментів між сесіями (журнали, CBT thought records, DBT diary cards, план безпеки) [CA].
Букінг наступної сесії (один клік для recurring).
Таймер: нагадування через 30 хв якщо нотатки не заповнені.
Автоматична генерація страхового клейму з даних сесії [CA].
Для клієнта:
Короткий фідбек (опціонально): «Чи відчуваєте ви, що вас почули?»
Підтвердження домашнього завдання.
Доступ до матеріалів сесії (якщо дозволено).
Пропозиція grounding-вправи (за маркером спеціаліста).
4.6 Надійність та обробка помилок у сесії
Втрата з'єднання: автоматичне перепідключення (до 30 сек), повідомлення іншому учаснику, fallback на аудіо.
Падіння сервісу: граціозна деградація — сесія продовжується без AI, буферизація на клієнті.
Мобільний додаток: background audio при згортанні, автоматичне перемикання Wi-Fi ↔ 4G/5G.
Індикатор якості зв'язку в реальному часі для обох учасників.

6. Страховий білінг та клінічна документація [CA]
Страховий білінг — гравітаційний центр платформи. Це одночасно топ-фіча, що приваблює терапевтів, основний механізм утримання, enabler для B2B/EAP контрактів та міст до outcomes-based pricing. Інвестиція у надійний claims workflow розблоковує каскадну цінність по всій платформі.
6.1 Workflow страхових клеймів [CA] [NEW]
Критична прогалина №1 з конкурентного аналізу. Без обробки клеймів неможливо обслуговувати 78%+ клієнтів зі страхуванням.
Подання 837P електронних клеймів
Авто-проводка ERA/835 (Remittance Advice)
Перевірка eligibility в реальному часі
Генерація superbill для клієнтів out-of-network
Верифікація copay перед сесією
Tracking denied claims з workflow повторного подання
Гарантована оплата спеціалісту незалежно від статусу клейму (опція для преміум-плану)
6.2 Підтримка крединшнелінгу [CA] [NEW]
Headway побудував бізнес на $2,3 млрд переважно на крединшнелінгу. Платформа підтримує:
Заявки на страхові панелі (подання, відстеження статусу)
Інтеграція з CAQH ProView (профілі провайдерів)
Крединшнелінг під груповим Tax ID платформи (опція)
Крединшнелінг під індивідуальним NPI спеціаліста
6.3 Шаблони клінічної документації [CA] [NEW]
Кожна EHR пропонує структуровані progress notes. AI-транскрипція недостатня — потрібні комплаєнс-шаблони для страхових аудитів та стандартів ліцензійних комісій.

25. Стратегічний висновок
Візія «операційної системи» для терапевтичної онлайн-практики є життєздатною саме тому, що жоден конкурент не покриває весь стек. SimplePractice не має маркетплейсу. BetterHelp не має управління практикою. Lyra та Spring Health — закриті екосистеми. Circle та Mighty Networks мають нульовий клінічний комплаєнс.
Три стратегічні принципи:
1. Страховий білінг — гравітаційний центр. Інвестиція у надійний claims workflow розблоковує каскадну цінність: пропозиція маркетплейсу, B2B-дохід, утримання спеціалістів, outcomes-based pricing.
2. AI обслуговує спеціаліста, не замінює його. Адміністративний AI (документація, білінг) прийнятний. Терапевтичний AI стикається з регуляторними бар'єрами. Позиціонування: AI як доповнення терапевтичних відносин.
3. Чотири інтеграційні шари створюють compound value: клінічна документація ↔ білінг ↔ вимірювання результатів ↔ інструменти залучення. Коли нотатки автоматично генерують клейми, оновлюють outcome tracking, тригерять міжсесійні завдання та інформують AI-matching — це мотт операційної системи, який жодне точкове рішення не може повторити.
Вікно для запуску модульної комплексної платформи звужується. Ринок консолідується: 104 M&A угоди у 2024-2025, Spring Health придбав Alma, 71% угод — venture-to-venture. Час діяти.
